## Fresh No-Credentials Runtime

The published npm tarball was unpacked, production dependencies were installed
inside that temp package directory, and the published CLI bundle was started
with a clean temp dataDir.

The runtime was launched with a preload guard that replaces `globalThis.fetch`
and blocks any non-local fetch. No blocked fetch was recorded, and no fetch
attempt log file was created during startup.

Startup log:

- `master cache refresh deferred until credentials are configured`
