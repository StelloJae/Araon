# Araon v1.1.0-beta.8

Status: published

Araon v1.1.0-beta.8 is the release candidate that should carry the post-beta.7
local main work into the public beta channel. It is needed because
`v1.1.0-beta.7` points at `148d61f`, while the accepted local product state is
after the chart/backfill, signal explanation, and beta acceptance commits.

- Adds deterministic signal explanations for surge/detail surfaces.
- Adds a dev-only local market simulator for UI validation.
- Keeps KIS credential setup live-only; the paper-trading setup option is
  removed because paper credentials are not useful for this read-only local beta.
- Adds persisted local candle history storage.
- Adds the `price_candles` SQLite table and candle repository/aggregation
  helpers.
- Adds the StockDetailModal `차트` tab.
- Adds TradingView Lightweight Charts rendering for local candle data.
- Supports chart intervals from local `1m` candles and daily aggregates,
  including `1D`, `1W`, and `1M`.
- Adds manual KIS daily historical backfill for daily candles.
- Records the single-ticker KIS daily live probe for `005930`.
- Records candle chart UI acceptance.
- Records chart/backfill MVP closeout.
- Records Araon beta acceptance as `CONDITIONAL GO`.
- Updates install/onboarding docs and beta.7 desktop asset names.

## Data / Runtime Policy

These notes describe the beta.8 release state. The later auto-operations default
promotion changes fresh settings to managed realtime/backfill after credentials
are configured.

- Raw tick persistence remains out of scope.
- Missing chart history is not synthesized.
- Fresh installs in beta.8 keep realtime OFF by default.
- The current local user data profile may have realtime enabled.
- Manual daily backfill is separate from full watchlist/background operation.
- Historical minute backfill is not included.

## Known Limitations

- DMG/EXE GUI manual validation is still pending.
- Full watchlist backfill remains HOLD.
- Background live backfill remains HOLD in beta.8.
- Historical minute backfill remains HOLD.
- Volume surge ratio requires enough local baseline samples.
- Desktop builds are unsigned.
- News/disclosure and observation memo/log features are not included.
