# Araon v1.1.4

Status: stable patch release for the localhost Araon monitor.

Araon v1.1.4 focuses on small usability fixes from real user feedback and one
modal data-quality bug found during live use.

## Highlights

- Windows sound notifications now include a WebAudio fallback for environments
  where the media-element beep does not play.
- Chart candles can be shown with either US-style colors or Korea-style colors
  from Settings.
- Stock detail metrics now include an EPS estimate when current price and PER
  are available.
- Favorite rows keep compact sparklines visible when local price history exists.
- The bottom status bar shows KST, KOSPI, KOSDAQ, USD/KRW, and WTI snapshot
  context.
- Stock detail quote metrics no longer lose REST detail fields after sparse
  WebSocket/SSE updates.
- The stock detail metric grid no longer exposes a gray trailing blank area
  when the last row is not full.

## Fix

- `PriceStore` now preserves REST detail fields such as open/high/low,
  market cap, PER/PBR, foreign ownership, and 52-week range when a later sparse
  realtime tick omits them.
- The client stock store applies the same carry-forward policy when an SSE
  reconnect snapshot omits detail fields.
- The carry-forward policy still respects explicit `null` values, so truly
  unavailable data is not fabricated.

## Known Limitations

- Windows sound fallback still depends on OS/browser audio device policy and
  should be checked on real Windows machines.
- Market summary values are best-effort public snapshots, not a trading feed.
- Desktop artifacts remain unsigned and may trigger Windows SmartScreen or
  macOS Gatekeeper warnings.
- Araon remains a read-only localhost monitoring tool with no order-entry
  feature.
