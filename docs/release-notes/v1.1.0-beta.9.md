# Araon v1.1.0-beta.9

Status: beta release

Araon v1.1.0-beta.9 carries the managed-defaults promotion after beta.8.
`v1.1.0-beta.8` remains unchanged; this release is the new public beta for the
credentials-after-first-run automatic operation policy.

## Managed Defaults

- After KIS credentials are configured, Araon manages integrated realtime quotes
  automatically with the cap40 H0UNCNT0 path.
- Daily historical candle backfill is managed automatically for tracked and
  favorite stocks.
- REST polling remains the fallback path.
- A clean install without credentials still makes zero external KIS calls.
- Explicitly persisted `false` settings are preserved as emergency-disabled
  operator intent.

Default fresh settings:

```txt
websocketEnabled=true
applyTicksToPriceStore=true
backgroundDailyBackfillEnabled=true
rateLimiterMode=live
```

## Safety And Operator Controls

- Realtime emergency disable remains available through the runtime operator
  route and Settings UI.
- Daily backfill can still be paused through persisted settings.
- Background daily backfill remains guarded:
  - no market-hours backfill
  - tracked/favorites scope only
  - no full-market/master backfill
  - no historical minute backfill
  - low-rate sequential execution
  - daily budget and cooldown guards
- Raw tick persistence remains out of scope.
- Missing chart data is not synthesized.

- Persisted false compatibility: verified.

- Backfill guard behavior: verified with mock/no-live checks.
- Existing local data UI smoke: not executed in that acceptance window to avoid
  unintentionally starting live runtime during market hours.

## Security

- Araon remains a localhost personal watchlist tool.
- Araon has no trading/order feature.
- Do not commit `credentials.enc`, local data directories, SQLite databases,
  runtime logs, tokens, approval keys, or account identifiers.
- Release validation includes a raw secret/token/key leak scan.

## Known Limitations

- Backfill budget/cooldown is still process-local; restart-safe persistence is a
  P1 follow-up before stable/latest promotion.
- Existing local data UI smoke should be repeated in a controlled window.
- DMG/EXE GUI install validation remains manual/pending if desktop assets are
  used for this beta.
- Full watchlist/background live backfill expansion remains HOLD beyond the
  guarded tracked/favorites policy.
- Historical minute backfill remains HOLD.
- Desktop builds are unsigned.
