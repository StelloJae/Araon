# Araon v1.1.0

Status: stable npm release for the localhost Araon monitor.

Araon v1.1.0 promotes the beta line to the default npm `latest` channel. This
release is intended for real personal use as a read-only Korean stock monitoring
tool. It does not add trading, order placement, or hosted-service behavior.

## Highlights

- Managed KIS realtime monitoring after credentials are configured.
- Integrated `H0UNCNT0` WebSocket realtime feed for up to 40 tracked/favorite
  tickers.
- REST polling fallback remains available when realtime is unavailable.
- Persistent intraday price history for same-day tick/sparkline continuity.
- Persisted local candle history and KIS daily candle backfill.
- Intraday chart intervals from stored local 1m candles.
- Daily, weekly, and monthly chart intervals from stored KIS daily candles.
- Selected-ticker today-minute backfill, guarded and not automatic watchlist
  wide minute backfill.
- Chart coverage auto-checks, data-quality display, and pinned candle
  inspection.
- Realtime surge and today-strength dashboards.
- Optional Telegram phone alerts with local delivery history.
- Cached news and disclosure feeds with optional Naver Search and OpenDART
  enrichment.
- Data-health diagnostics for backfill, candle retention, news/disclosure
  cache, signal history, and notification delivery.
- Dev simulator is no longer always visible in the normal user flow.
- Observation plan, observation memo, and observation timeline surfaces were
  removed to keep the modal focused on monitoring.

## First-Run Safety

- Fresh installs have no credentials and make no external KIS calls.
- `POST /master/refresh` is blocked until credentials are configured.
- After credentials are configured, Araon manages realtime quotes and guarded
  daily backfill automatically.
- Araon stores credentials locally and encrypted in the selected data directory.
- Araon has no order-entry or trading feature.

## Install

Run directly from npm:

```bash
npx @stellojae/araon
```

Or install globally:

```bash
npm install -g @stellojae/araon
araon
```

## Optional Providers

Araon works without optional provider keys, but can enrich the UI when these are
configured locally:

- Naver Search API for news search results.
- OpenDART API for disclosure feed enrichment.
- Telegram bot token and chat ID for phone alerts.

Do not commit `.env`, credentials, SQLite data, logs, or runtime state.

## Known Limitations

- Desktop artifacts are unsigned and may trigger macOS Gatekeeper or Windows
  SmartScreen warnings.
- Apple Developer ID signing and notarization are not done.
- Windows desktop execution remains manual-validation pending.
- Full watchlist minute backfill remains prohibited.
- Automatic/background historical minute backfill remains prohibited.
- News/disclosure enrichment depends on provider availability and quotas.
- This is a localhost personal tool, not a hosted multi-user service.
