# Araon v1.1.0-beta.7

Araon v1.1.0-beta.7 is a first-run guidance and onboarding polish beta. It
does not add trading features, change realtime defaults, or promote npm
`latest`.

## Install

During the beta period, use the explicit npm beta tag:

```bash
npx @stellojae/araon@beta
```

Or install globally:

```bash
npm install -g @stellojae/araon@beta
araon
```

## Changes

- Added an install acceptance matrix for the v1.1.0-beta.6 distribution paths.
- Clarified README and INSTALL first-run guidance for npm, npx, global CLI,
  GitHub Release tarball/source archive, and unsigned desktop beta users.
- Clarified that first-run users need their own KIS OpenAPI credentials.
- Clarified that Araon is read-only and has no order/trading feature.
- Clarified that fresh installs keep realtime OFF and REST polling remains the
  fallback path.
- Clarified CLI, source, and desktop data-directory behavior.
- Added first-run setup copy to the KIS credentials setup screen.
- Corrected older `autoSector` wording to the current KIS official index
  industry grouping policy.

## Known Limitations

- DMG/EXE GUI install acceptance still needs manual validation.
- Desktop beta artifacts are unsigned, so macOS Gatekeeper and Windows
  SmartScreen warnings are expected.
- Fresh installs keep realtime OFF until the user opts in.
- Volume-surge ratios appear only after enough local same-session/time-bucket
  baseline samples exist.
- Signed desktop release is not available yet.
- Docker is not included.
