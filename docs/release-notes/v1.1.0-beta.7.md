# Araon v1.1.0-beta.7 Draft

Araon v1.1.0-beta.7 is planned as a first-run acceptance and onboarding polish
release. It does not add trading features or change realtime defaults.

## Planned Changes

- Add an install acceptance matrix for the v1.1.0-beta.6 beta distribution
  paths.
- Clarify the npm beta quick start:
  `npx @stellojae/araon@beta`.
- Clarify that first-run users need their own KIS OpenAPI credentials.
- Clarify that Araon is read-only and has no order/trading feature.
- Clarify that fresh installs keep realtime OFF and REST polling remains the
  fallback path.
- Clarify CLI, source, and desktop data-directory behavior.
- Add first-run setup copy to the credentials setup screen.

## Executed In This Pass

- `npx @stellojae/araon@beta --help`
- `npx @stellojae/araon@beta --version`
- `npx @stellojae/araon@beta --no-open --port 43910 --data-dir <temp>`
- `npm install -g @stellojae/araon@beta`
- `araon --help`
- `araon --version`
- `araon --no-open --port 43911 --data-dir <temp>`
- GitHub Release npm tarball install into a temp npm prefix.
- GitHub Release source archive download and metadata inspection.

## Not Executed

- Real KIS credential entry.
- Authenticated live KIS REST quote calls.
- KIS WebSocket sessions.
- Cap tests.
- macOS DMG launch.
- Windows EXE launch.

## Known Limitations

- Desktop beta artifacts are unsigned. macOS Gatekeeper and Windows SmartScreen
  warnings are expected.
- npm `latest` is not promoted in this release.
- Docker is not included.
- Volume-surge ratios still require local same-session/time-bucket baseline
  samples before a ratio is shown.
