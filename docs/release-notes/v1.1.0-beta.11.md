# Araon v1.1.0-beta.11

Status: beta patch release, `CONDITIONAL GO`

Araon v1.1.0-beta.11 is a narrow patch release for the clean first-run startup
policy. It does not add product features and does not update `npm latest`.

## Patch Summary

- Clean no-credentials startup now performs zero external KIS fetches.
- Public KIS master refresh is deferred until credentials are configured.
- `POST /master/refresh` now requires credentials and returns
  `MASTER_REFRESH_REQUIRES_CREDENTIALS` before setup.
- After successful credentials registration, Araon starts the same non-blocking
  master refresh hook so dashboard rendering remains unblocked.
- A regression test now starts the server with a clean temp dataDir and fails if
  any external `fetch` happens before credentials.

## What Was Found In beta.10

The v1.1.0-beta.10 post-release acceptance confirmed:

- Credentialed KIS token issuance: 0
- Approval key issuance: 0
- WebSocket/cap tests: 0
- Daily/minute backfill live runs: 0
- Raw credential/token/account output: 0

However, beta.10 did trigger the public KIS KOSPI/KOSDAQ master file refresh on
clean no-credentials startup. That was not a credential leak, but it violated
the product policy that fresh installs should make no external KIS calls before
credentials exist.

## Security

- Araon has no trading or order-entry feature.
- Do not commit `credentials.enc`, local data directories, SQLite databases,
  runtime logs, tokens, approval keys, or account identifiers.
- The package contents audit must exclude local runtime state and secrets.

## Known Limitations

- Release status is `CONDITIONAL GO`, not stable/latest.
- `npm latest` is not updated.
- DMG/EXE GUI manual validation remains pending for this patch release.
- Existing local data live UI smoke should still be repeated in a controlled
  window when useful.
- Full watchlist minute backfill remains HOLD.
- Automatic/background historical minute backfill remains HOLD.
- Desktop builds remain unsigned.
