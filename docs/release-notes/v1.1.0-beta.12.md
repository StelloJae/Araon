# Araon v1.1.0-beta.12

Status: beta desktop patch release, `CONDITIONAL GO`

Araon v1.1.0-beta.12 is a narrow desktop packaging patch for the macOS beta
artifacts. It does not add product features, does not update `npm latest`, and
does not change the localhost-only read-only product policy.

## Patch Summary

- macOS Electron Builder packaging now uses ad-hoc signing
  (`mac.identity="-"`) instead of skipping signing entirely.
- Fresh local macOS DMG and ZIP artifacts now contain an `Araon.app` bundle that
  passes `codesign --verify --deep --strict`.
- macOS desktop packaging now uses the legacy native dependency rebuilder so
  `better-sqlite3` is packaged for Electron ABI 145, not the host Node ABI.
- The previous broken resource-seal error is removed:

```txt
code has no resources but signature indicates they must be present
```

- `spctl -a -vv` still rejects the app because the desktop beta is not Apple
  Developer ID-signed or notarized. That is expected for this beta.
- The npm CLI path is unchanged except for the version bump.

## Desktop Posture

macOS beta artifacts are now ad-hoc signed, but not notarized:

```txt
codesign --verify --deep --strict: pass
spctl -a -vv: rejected, expected without Developer ID/notarization
better-sqlite3 native ABI: Electron 41 / NODE_MODULE_VERSION 145
```

Windows artifacts remain unsigned and still require manual execution validation
on Windows.

## First-Run Policy

The first-run safety policy remains unchanged:

- Fresh installs have no credentials.
- No external KIS calls should occur before credentials are configured.
- `POST /master/refresh` is blocked before credentials.
- After credentials are configured, Araon manages realtime quotes and guarded
  daily backfill automatically.
- Araon has no trading or order-entry feature.

## Known Limitations

- Release status is `CONDITIONAL GO`, not stable/latest.
- `npm latest` is not updated.
- macOS browser-downloaded quarantine/Finder launch path remains conditional.
- Apple Developer ID signing and notarization are not done.
- Windows Setup/portable EXE execution remains manual pending.
- Full watchlist minute backfill remains HOLD.
- Automatic/background historical minute backfill remains HOLD.
