# Araon v1.2.0

Status: release candidate for the Toss-first product 100 lane.

Araon v1.2.0 consolidates the product work that moved the app from a
KIS-first local monitor toward a Toss-first watchlist, chart, account, and
agent decision-support dashboard. The release remains read-only: it does not
place, cancel, amend, or automate orders.

## Highlights

- Toss-first TOP100, search, quote, chart, watchlist, account, and portfolio
  surfaces are now the primary product path.
- Favorites are aligned around Toss watchlist and holdings, with local
  favorites kept as fallback/cache state.
- Product identity, quote key, and chart key handling were separated so Toss
  product codes and six-digit KRX tickers do not leak into the wrong routes.
- Favorites and holdings rows show real-data price, direction, percent, product
  icons, and sparkline/history surfaces when available.
- Mini/full chart views use Toss historical candle data plus local cache and a
  live quote overlay for the active candle.
- KIS is contained as an optional low-latency realtime tracking rail, not the
  account, watchlist, chart, or TOP100 source of truth.
- The Toss account rail now supports denser product rows, product icons, hover
  affordance, and row-click chart changes.
- Agent views were upgraded into decision-support surfaces with detected
  candidates, reasons, simulated previews, risk/audit context, readiness gaps,
  and an explicit live-trading lock.
- CLI/package paths remain aligned with `araon`, `araon doctor`, `araon
  status`, `araon open`, and guarded reset commands.

## UX And Documentation

- README and installation copy now describe the Toss-first product path and
  optional KIS realtime rail accurately.
- Korean README was rewritten to remove stale KIS-first onboarding language.
- Product 100 visual QA artifacts are archived under
  `docs/archive/visual-qa/2026-05-19-product-100/`.
- Product readiness and completion audits are kept under `docs/research/`.

## Known Boundaries

- Araon remains a local read-only monitoring and decision-support tool.
- Live order execution, order cancellation/amendment, account mutation, and
  live auto-trading are intentionally locked.
- Agent live-trading readiness means the locked pre-execution lane is modeled;
  it does not mean a live trading bot is enabled.
- Desktop artifacts remain unsigned and may trigger Windows SmartScreen or
  macOS Gatekeeper warnings.
- Market data depends on upstream Toss/KIS/Naver/OpenDART/Telegram availability
  and policy limits.
